# Software Requirements

## Problem Statement

It is difficult to post on linkedin every day to increase user engagement. There is a need to automate thsi process but keep the human int he loop by providing current events and taking user input prior to suggesting linkedin psot content and askign for human aprooval before posting.

## Functional Requirments 

1. The system shall allow users to create an account for this software application.
2. The system shall connect to the user's linkedin account through the linkedin API to allow posting.
3. The system shall take as input the subject categories or specialties the user is interested in and tailor the content to these interests.
4. The system shall allow the user to update interests in the settings.
5. The system shall have a login and authentication page.
6. The system shall allow users to log in tot heir account.
7. The system shall allow users to create an account with a username, password, and linkedin credentials to be used for linkedin posting.
8. The system shall keep track of user accounts in a secure manner such as encryption plus Salting.
9. The system shall use a Free and appropriate database to store user authentication, usernames, passwords, and sessions.
10. The system shall use a Free and appropriate database to store the user's calendar of scheduled posts.
11. The system shall provide a dashboard that is viewable on desktop and mobile upon user login.
12. The system shall use a Free source of generative AI sufficient to write linkedin posts that cotnain text and emoji's
13. The system shall use a Free source of web scraping to get the latest current events in the subject of interest to the user
14. The system shall take in two modes of linkedin post generation
15. The system shall provide an first option to create a novel linkedin post based on a specific latest topic in the field of interest. Whicht he system will provide 3 sample articles or topics of latest trends in their subject of interest the user can select from. A post will be generated once selected. The user will be able to aproove the post or have the artificial intelligence regenerate that post or allow the user to go back to the screen where 3 new optiosn are provided
16. The system shall provide a second option to allow the user to write about what they would like to post about and provide links or other context and the system will create a linkedin post with the user's provided context. The user will be able to aproove the post or have the artificial intelligence regenerate that post.
17. The system shall use generative artificial intelligence to write a linkedin post in linkedin type of language and styling aimed at getting user attention and maximizing impressions, likes, and views.
18. The system shall allow the user to modify and edit the linkedin post in an editable text box on the user interface.
19. The system shall allow approved posts to be posted now or scheduled to be posted on linkedin.
20. The system shall allow users to see a calendar user interface that shows when the posts are scheduled.
21. The system shall allow users to log out.
22. The system shall allow users to delete their account, whichw illr emove all their user specific data from the database.
